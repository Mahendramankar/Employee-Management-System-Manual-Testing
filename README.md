# Employee Management System – Manual Testing Project

## Project Overview
This manual testing project is based on an Employee Management System application.
The project focuses on validating the Add User functionality using SRS-based testing.

## Modules Covered
- Add User Page
- Form Field Validations
- Submit and Cancel Button Behavior

## Testing Scope
- Requirement Analysis (SRS)
- Test Scenario Preparation
- Test Case Design
- Functional Testing
- Boundary Value Analysis
- Negative Testing
- Input Validation Testing

## Validations Tested
- Username (6–32 alphabets only)
- Password (6–32 characters, special characters allowed)
- Designation (Dropdown mandatory)
- Email ID format validation
- Mobile number (10-digit numeric validation)
- Gender selection (Mandatory)
- Date of Birth (Valid date, format, no future date)
- Address length validation (20–200 characters)
- Submit & Cancel button functionality
